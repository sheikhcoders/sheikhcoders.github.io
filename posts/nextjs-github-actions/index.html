Deploy Next.js with GitHub Actions

GitHub Actions can build, test, and deploy your Next.js project on every push with caching, quality gates, and a final deploy hook.

গিটহাব অ্যাকশন্স আপনার প্রতিটি পুশের পর নেক্সট.জেএস প্রজেক্ট বিল্ড, টেস্ট ও ডিপ্লয় করতে পারে—ক্যাশ, কোয়ালিটি গেট ও ডিপ্লয় হুকসহ।

  • GitHub Actions
  • Next.js Deploy
  • CI/CD

When you push to main, the workflow checks out code, installs dependencies with caching, runs tests, builds the production bundle, and finally triggers your deployment target. 

name: Deploy Next.js

on:
  push:
    branches: ["main"]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      deployments: write
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: "npm"
      - run: npm ci
      - run: npm run lint
      - run: npm run test -- --watch=false
      - run: npm run build
      - name: Trigger Vercel Deploy Hook
        env:
          VERCEL_DEPLOY_HOOK: ${{ secrets.VERCEL_DEPLOY_HOOK }}
        run: curl -X POST "$VERCEL_DEPLOY_HOOK"

Store VERCEL_DEPLOY_HOOK as a repository secret. For static exports, replace the final step with actions/upload-pages-artifact followed by actions/deploy-pages to target GitHub Pages.

  • Use reusable workflows for linting so multiple projects inherit the same checks.
  • Protect main with required reviews and status checks before deployments run.
  • Log workflow run durations to spot slow npm installs or failing steps quickly.

main ব্রাঞ্চে পুশ করলেই ওয়ার্কফ্লো কোড চেকআউট, ক্যাশসহ ডিপেন্ডেন্সি ইনস্টল, টেস্ট চালানো, প্রোডাকশন বিল্ড তৈরি এবং শেষে ডিপ্লয় হুক ট্রিগার করে।

উপরের উদাহরণে VERCEL_DEPLOY_HOOK রিপোজিটরি সিক্রেট হিসেবে সংরক্ষণ করুন। স্ট্যাটিক এক্সপোর্ট ব্যবহার করলে শেষ ধাপটি actions/upload-pages-artifact এবং actions/deploy-pages দিয়ে প্রতিস্থাপন করুন যাতে গিটহাব পেজেসে প্রকাশ করা যায়।

  • একই লিন্টিং নিয়ম সব প্রজেক্টে প্রয়োগ করতে রিইউজেবল ওয়ার্কফ্লো ব্যবহার করুন।
  • ডিপ্লয়ের আগে main ব্রাঞ্চে আবশ্যিক রিভিউ এবং স্ট্যাটাস চেক কনফিগার করুন।
  • ওয়ার্কফ্লো রান টাইম লগ রাখলে ধীর npm ইনস্টল বা ব্যর্থ স্টেপ দ্রুত শনাক্ত করা যায়।
