Query String Deep Dive (English & Bangla)

A URL query string starts after a ? and carries key-value pairs such as utm_source=newsletter. These parameters help filter content, power search experiences, and pass analytics insights without changing the core page.

একটি ইউআরএল কুয়েরি স্ট্রিং ? চিহ্নের পরে শুরু হয় এবং utm_source=newsletter এর মতো কী-ভ্যালু জোড়া বহন করে। এসব প্যারামিটার কনটেন্ট ফিল্টার করতে, সার্চের ফল উন্নত করতে এবং মূল পৃষ্ঠাটি অপরিবর্তিত রেখে অ্যানালিটিক্স তথ্য পাঠাতে সহায়তা করে।

Focused SEO Checklist

  • Use readable keys (category=css) so crawlers understand content groupings.
  • ডাইনামিক প্যারামিটারকে গুগল সার্চ কনসোলে ডিফাইন করে ডুপ্লিকেট কনটেন্টের ঝুঁকি কমান।
  • Pair canonical URLs with query variants to preserve ranking signals.
  • অ্যানালিটিক্স ট্র্যাকিং (যেমন utm_) শেষ হলে অপ্রয়োজনীয় প্যারামিটার অপসারণ করুন।

Logic Flow for Dynamic Pages

  1. Parse the query string with built-in browser APIs like URLSearchParams.
  2. ফিল্টার প্রয়োগের আগে ডিফল্ট মান সেট করে মোবাইল ব্যবহারকারীর জন্য দ্রুত লোড নিশ্চিত করুন।
  3. Cache popular combinations to keep mobile latency low and UX focused.

Build a Clean Portfolio Landing Page

Start with semantic containers (<header>, <main>, <footer>) so screen readers and search engines map the layout without guesswork. Keep the hero section concise with a primary CTA and a muted secondary link for recruiters who prefer quick scans.

  • Use fluid typography via clamp() to maintain readability from 320px to desktop screens.
  • Compress preview imagery with <picture> and loading="lazy" attributes to stay under 100 KB.
  • Ship Open Graph and Twitter meta tags that match your hero copy for consistent sharing.

শুরুতেই সেমান্টিক এলিমেন্ট (<header>, <main>, <footer>) ব্যবহার করলে স্ক্রিন রিডার এবং সার্চ ইঞ্জিন লেআউট সহজে বুঝতে পারে। হিরো সেকশনে একটি প্রধান কল-টু-অ্যাকশন এবং দ্রুত স্ক্যান পছন্দ করা রিক্রুটারদের জন্য একটি সেকেন্ডারি লিঙ্ক রাখুন।

  • clamp() টাইপোগ্রাফি ব্যবহার করে ৩২০ পিক্সেল থেকে ডেস্কটপ পর্যন্ত পাঠযোগ্যতা বজায় রাখুন।
  • <picture> ও loading="lazy" দিয়ে প্রিভিউ ইমেজ ১০০ কিলোবাইটের মধ্যে সংকুচিত রাখুন।
  • ওপেন গ্রাফ ও টুইটার মেটা ট্যাগে হিরো কপির সঙ্গে সামঞ্জস্যপূর্ণ বার্তা দিন যাতে শেয়ারের সময় ভিজুয়াল এক থাকে।

Mastering URL Query Strings for Focused SEO

When you expose filters or campaign parameters, document them in your analytics playbook and Search Console settings. This prevents duplicate crawling and keeps reports aligned with marketing goals.

  1. Whitelist marketing keys like utm_medium, campaign, and ref in canonical logic.
  2. Reject untrusted parameters server-side to block cache poisoning and reduce bundle re-renders.
  3. Log query combinations and sort by conversions to prune low-impact URLs every quarter.

ফিল্টার বা ক্যাম্পেইন প্যারামিটার ব্যবহার করলে অ্যানালিটিক্স প্লেবুক এবং সার্চ কনসোল সেটিংসে স্পষ্ট করে উল্লেখ করুন। এতে ডুপ্লিকেট ক্রলিং কমে এবং রিপোর্ট মার্কেটিং লক্ষ্য অনুযায়ী থাকে।

  1. ক্যানোনিকাল সেটআপে utm_medium, campaign, ref এর মতো মার্কেটিং কী প্রাধান্য দিন।
  2. সার্ভার সাইডে অবিশ্বস্ত প্যারামিটার বাতিল করুন যাতে ক্যাশ পয়জনিং ও অপ্রয়োজনীয় রি-রেন্ডার কমে।
  3. প্রতি কোয়ার্টারে কোয়েরি কম্বিনেশন লগ বিশ্লেষণ করে কম পারফর্ম করা ইউআরএল কেটে দিন।

Starter Guide to Next.js Performance

Embrace the app/ router for automatic code-splitting and metadata APIs. Add next/font for critical font inlining and use Image for responsive asset delivery without cumulative layout shift.

  • Enable experimental.optimizePackageImports to trim unused library code in production.
  • Measure Core Web Vitals via reportWebVitals and push results to your preferred analytics endpoint.
  • Create a Lighthouse CI budget (largest contentful paint < 2.5s, total blocking time < 200ms) before launch.

স্বয়ংক্রিয় কোড স্প্লিটিং ও মেটাডাটা এপিআই পেতে app/ রাউটার ব্যবহার করুন। next/font দিয়ে ক্রিটিকাল ফন্ট ইনলাইন করুন এবং Image কম্পোনেন্ট ব্যবহার করলে রেস্পন্সিভ ইমেজ সহজে পরিবেশন হয়।

  • প্রোডাকশনে অব্যবহৃত লাইব্রেরি কোড কমাতে experimental.optimizePackageImports সক্রিয় করুন।
  • reportWebVitals ব্যবহার করে কোর ওয়েব ভাইটালস মাপুন এবং পছন্দের অ্যানালিটিক্স এন্ডপয়েন্টে পাঠান।
  • লাইভ করার আগে লাইটহাউস সি আই বাজেট সেট করুন (LCP < ২.৫ সেকেন্ড, TBT < ২০০ মিলিসেকেন্ড)।

AI Writing Workflow for Developers

Draft outlines with prompts that specify reader level, tone, and context. Iterate by reviewing AI suggestions against your project requirements, then store final snippets in a shared knowledge base.

  • Tag AI-generated changes in version control for transparent peer reviews.
  • Keep a reusable prompt library for changelog updates, release notes, and onboarding guides.
  • Automate grammar checks with CI jobs so documentation stays consistent across repos.

পাঠকের স্তর, টোন ও কনটেক্সট উল্লেখ করে প্রম্পট তৈরি করলে এআই আউটলাইন স্পষ্ট হয়। প্রকল্পের প্রয়োজনীয়তার সাথে মিলিয়ে পরামর্শ যাচাই করুন এবং চূড়ান্ত নোট শেয়ার্ড নলেজ বেসে সংরক্ষণ করুন।

  • পিয়ার রিভিউ সহজ করতে এআই-জেনারেটেড পরিবর্তন ভার্সন কন্ট্রোলে আলাদা ট্যাগ দিন।
  • চেঞ্জলগ, রিলিজ নোট ও অনবোর্ডিং গাইডের জন্য পুনঃব্যবহারযোগ্য প্রম্পট লাইব্রেরি রাখুন।
  • ডকুমেন্টেশন ধারাবাহিক রাখতে সি আই জবের মাধ্যমে গ্রামার চেক অটোমেট করুন।

Deploy Next.js with GitHub Actions

GitHub Actions can build, test, and deploy your Next.js project whenever you push to the main branch. The workflow below installs dependencies with caching, runs unit tests, builds the production bundle, and publishes to Vercel via a deploy hook. 

name: Deploy Next.js

on:
  push:
    branches: ["main"]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      deployments: write
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: "npm"
      - run: npm ci
      - run: npm run lint
      - run: npm run test -- --watch=false
      - run: npm run build
      - name: Trigger Vercel Deploy Hook
        run: curl -X POST "$VERCEL_DEPLOY_HOOK"

Store VERCEL_DEPLOY_HOOK as an encrypted repository secret. If you publish static exports, replace the last step with a actions/upload-pages-artifact and actions/deploy-pages pair targeting GitHub Pages.

যখনই আপনি মেইন ব্রাঞ্চে পুশ করবেন, গিটহাব অ্যাকশন্স নেক্সট.জেএস প্রজেক্টের ডিপেন্ডেন্সি ইনস্টল, টেস্ট রান এবং বিল্ড তৈরি করতে পারে। নিচের ওয়ার্কফ্লোটি ক্যাশ সহ ডিপেন্ডেন্সি ইনস্টল, ইউনিট টেস্ট, প্রোডাকশন বিল্ড এবং ভার্সেল ডিপ্লয় হুক ট্রিগার করে।

VERCEL_DEPLOY_HOOK সিক্রেট হিসেবে সংরক্ষণ করুন। যদি স্ট্যাটিক এক্সপোর্ট ব্যবহার করেন তবে শেষ ধাপে গিটহাব পেজেসের জন্য actions/upload-pages-artifact এবং actions/deploy-pages স্টেপ ব্যবহার করুন।

  • নিজস্ব ডোমেইন সংযোগের আগে প্রিভিউ পরিবেশে বিল্ড চেক করুন।
  • ডিপ্লয়মেন্ট ব্যর্থ হলে workflow_run ইভেন্ট দিয়ে রোলব্যাক ট্রিগার করতে পারেন।
  • আরো নিরাপত্তার জন্য এনভায়রনমেন্ট প্রোটেকশন রুলস ও রিভিউয়ার অ্যাপ্রুভাল যুক্ত করুন।
